# AI API Gateway 项目记忆

## 项目基本信息
- 仓库: https://github.com/0311yet/ai_api.git
- 技术栈: Python 3.10+ / FastAPI / SQLite (aiosqlite) / SQLAlchemy 异步
- 端口: 8080 (Linux默认) / 8088 (Windows测试)
- 启动: `python -m uvicorn app.main:app --host 0.0.0.0 --port 8080`

## 已实现功能
- ✅ OpenAI 兼容输入格式 (/v1/chat/completions, /v1/models)
- ✅ 多厂商接入: OpenAI / Anthropic / Google / 自定义兼容端点
- ✅ 格式转换: OpenAI ↔ Anthropic 自动转换 (含SSE流式)
- ✅ 模型路由: 按模型名匹配通道 + 默认兜底通道
- ✅ 模型池: 多Key权重负载均衡 + 故障转移 + 错误计数/封禁
- ✅ 监控: 请求数/成功失败/延迟/TTFT/Token总量/7天30天趋势
- ✅ 请求历史: 输入输出Token/状态/错误/请求体
- ✅ Web管理面板: 仪表盘/通道CRUD/密钥池/模型统计/请求历史

## 关键坑点 (已踩过)
1. **SQLAlchemy异步懒加载**: `len(ch.keys)` 在异步session会触发MissingGreenlet错误
   → 修复: 用 `selectinload(Channel.keys)` 预加载, 避免循环中访问关系属性
2. **Jinja2模板路径**: web.py 用 `__file__` 算路径时写成了 `__file__/templates`
   → 修复: `Path(__file__).resolve().parent.parent / "web" / "templates"`
3. **中文/emoji编码**: print()含中文/emoji在Windows GBK环境会崩溃
   → 修复: 用ASCII字符串
4. **time模块未导入**: format_converter.py用了time但忘记import
   → 修复: 加了 `import time`
5. **estimate_tokens逻辑错**: 原先 `for char in range(output_chars)` 越界
   → 修复: 重写为 count_text_tokens() 函数
6. **Windows端口8080被屏蔽**: permission denied
   → 换端口8088测试, Linux部署用8080
7. **git rebase冲突**: 远端有README.md (只有1行), 本地完整README冲突
   → 用 `git checkout --ours README.md` 保留本地版本
